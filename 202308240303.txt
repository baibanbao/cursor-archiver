错误信息指出，“从 2021 年 8 月 13 日起，已经移除了对密码认证的支持。” 这意味着你不能再使用你的 GitHub 用户名和密码来进行认证。相反，GitHub 现在要求使用个人访问令牌（PATs）或 SSH 密钥。

这是使用个人访问令牌进行认证的步骤：

登录到你的 GitHub 账户。

点击你的头像，然后选择 "Settings"。

在左侧菜单中选择 "Developer settings"。

点击 "Personal access tokens"。

点击 "Generate new token"。在 "Note" 字段中输入一个描述（例如 "Git command line access"）。然后在 "Select scopes" 部分，至少选择 "repo" 来给予此 token 对你私有仓库的访问权限。

点击 "Generate token"。

你将会看到一个新的 token。确保复制此 token - 你将无法再次查看它。

然后，当你再次运行 git push -u origin master 时，将会提示你输入用户名和密码。在这里，用户名是你的 GitHub 用户名，而密码应输入你刚刚创建的个人访问令牌。这应该解决认证失败的问题。
